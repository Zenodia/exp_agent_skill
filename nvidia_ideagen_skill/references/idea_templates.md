# Idea Generation Templates

Pre-defined templates for different idea generation scenarios.

## Template Categories

### 1. Product Idea Template

**Structure:**
```
## [Product Name]

### Core Concept
[1-2 sentence description]

### Problem Solved
[What pain point does this address?]

### Target Users
[Who is this for?]

### Key Features
1. [Feature 1]
2. [Feature 2]
3. [Feature 3]

### Unique Value Proposition
[What makes this different/better?]

### Potential Impact
[How does this improve users' lives?]

### Next Steps
- [ ] [First action item]
- [ ] [Second action item]
```

**Example:**
```
## SmartGrocery

### Core Concept
An AI-powered grocery shopping assistant that learns your preferences and auto-generates shopping lists based on your meal plans and inventory.

### Problem Solved
- Time spent planning meals and making lists
- Forgetting essential items
- Food waste from poor planning

### Target Users
- Busy professionals
- Families with recurring meal patterns
- Health-conscious shoppers

### Key Features
1. AI meal planning with dietary preferences
2. Automatic inventory tracking via receipt scanning
3. Smart list generation with store layout optimization
4. Recipe suggestions based on available ingredients

### Unique Value Proposition
Combines meal planning, inventory management, and smart shopping in one seamless experience.

### Potential Impact
Save users 2-3 hours per week on meal planning and reduce food waste by 30%.

### Next Steps
- [ ] Develop MVP with basic meal planning
- [ ] Build receipt scanning prototype
- [ ] Test with 50 beta users
```

## Prompt Templates for AI Generation

### General Idea Generation
```
Generate {num_ideas} innovative ideas for {topic}.

Context: {context}
Target audience: {audience}
Constraints: {constraints}

For each idea, provide:
1. Title
2. Description (2-3 sentences)
3. Key features or components
4. Potential impact
5. Next steps to explore
```

### Problem-Solution Ideation
```
Problem: {problem_description}

Generate {num_ideas} solutions that:
- Address the core pain point
- Are feasible to implement
- Have clear value proposition
- Consider {constraints}

Format each solution with:
- Solution name
- How it solves the problem
- Key differentiators
- Implementation approach
```

### Creative Variation
```
Original idea: {original_idea}

Generate {num_ideas} creative variations by:
- Changing the target audience
- Modifying the delivery method
- Adding unexpected features
- Combining with other concepts

For each variation, explain:
- What changed
- Why this might be better/different
- New opportunities created
```

### Domain-Specific Ideation
```
Domain: {domain_name}
Focus area: {focus_area}
Constraints: {constraints_list}

Generate {num_ideas} concepts that:
- Are innovative within this domain
- Address specific pain points
- Leverage emerging technologies
- Are feasible given constraints

Include for each:
- Concept name
- Core innovation
- Target users
- Implementation considerations
```

## Output Format Templates

### Structured List Format
```
# Ideas: {topic}

## Idea 1: [Title]

**Description:** [2-3 sentences]

**Key Features:**
- Feature 1
- Feature 2
- Feature 3

**Impact:** [Why this matters]

**Next Steps:**
- Step 1
- Step 2

---

[Repeat for each idea]
```

### Comparison Table Format
```
# Ideas: {topic}

| Idea | Description | Pros | Cons | Feasibility |
|------|-------------|------|------|-------------|
| 1. [Title] | [Brief desc] | [Advantages] | [Limitations] | [High/Med/Low] |
| 2. [Title] | [Brief desc] | [Advantages] | [Limitations] | [High/Med/Low] |
```

### Mind Map Format
```
# Ideas: {topic}

Central Concept: {topic}

Branch 1: [Category 1]
├── Idea 1.1
│   ├── Feature A
│   └── Feature B
└── Idea 1.2
    ├── Feature C
    └── Feature D

Branch 2: [Category 2]
├── Idea 2.1
└── Idea 2.2
```

## Tips for Using Templates

1. **Customize**: Adapt templates to your specific needs
2. **Iterate**: Start with template, then expand
3. **Mix and Match**: Combine elements from different templates
4. **Stay Flexible**: Templates are guides, not constraints
5. **Document**: Save successful variations as new templates
